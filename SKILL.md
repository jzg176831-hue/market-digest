# 金融市场日报（market-digest）— 运行手册

> 安装步骤请见 **INSTALL.md**，本文件只描述运行逻辑。

---

## 〇、输出处理（所有触发方式通用）⚠️ 必读

命令执行完成后，在 stdout 中找到如下标记块：

```
__SKILL_REPLY_START__
<报告正文>
__SKILL_REPLY_END__
```

**处理规则（严格遵守）：**

1. 提取两个标记之间的全部文本。
2. **原封不动**输出给用户——不得总结、不得改写、不得截断、不得加任何前缀或后缀。
3. 如果 stdout 中没有这两个标记，说明命令出错或日报为空，按第四节"异常处理"操作。

> **错误示范**："日报已生成，内容如下：银行业绩……（总结）"
> **正确示范**：直接把标记之间的完整 Markdown 文本贴出来，一字不改。

## 一、定时任务触发（自动执行，无需判断）

| 定时任务 | 命令 | 说明 |
|----------|------|------|
| `0 8 * * *` 每天 8:00 | `node daily-full.js` | 完整日报：抓取 + 聚类 + 生成报告 |
| `0 0-7,9-23 * * *` 每小时 | `node fetch.js` | 仅抓取文章写入数据库，不生成报告 |

定时任务触发时，直接按上表运行对应命令，不需要询问用户，完成后将输出推送到配置的渠道。

---

## 二、用户主动触发（需要时间判断）

用户说"出日报"、"财经日报"、"早报"、"今天/今日财经"等话语时，**先看当前时间**再决定运行哪个命令：

### 情况 A：用户要"今天的/今日的/早报"等

```
当前时间 < 9:30  →  node daily.js
                     （昨日 08:00 → 今日 08:00 的文章，即标准早报内容；仅基于数据库生成，不额外抓取）

当前时间 ≥ 9:30  →  node daily.js today
                     （今日 08:00 → 当前时间的文章，包含上午最新资讯；仅基于数据库生成，不额外抓取）
```

### 情况 B：用户要"昨天的/昨日的"日报

```
任何时间  →  node daily.js
             （昨日 08:00 → 今日 08:00，标准昨日日报；仅基于数据库生成，不额外抓取）
```

### 情况 C：用户要某个具体日期的日报（如"4月6号的日报"）

```
node daily.js 2026-04-06
```

### 情况 D：用户只要抓取文章，不需要生成日报

```
node fetch.js
```

## 三、命令参考

| 命令 | 文章时间范围 | 适用场景 |
|------|-------------|----------|
| `node daily-full.js` | 昨日 08:00 → 今日 08:00 | 8点定时任务：先抓取，再生成日报 |
| `node daily.js` | 昨日 08:00 → 今日 08:00 | 9:30前用户要今日 / 用户要昨日；仅基于数据库生成 |
| `node daily.js today` | 今日 08:00 → 当前时间 | 9:30后用户要今日 |
| `node daily.js YYYY-MM-DD` | 该日 08:00 → 次日 08:00 | 用户指定日期 |
| `node fetch.js` | 当前增量抓取 | 仅抓取，不生成报告 |

工作目录：技能所在目录（安装时由 Agent 确定，参见 INSTALL.md 第 1 步）

## 四、异常处理

- **DB 未配置**：输出提示"数据库未配置，请先运行安装流程（参见 INSTALL.md）"
- **Chromium 启动失败**：提示用户安装系统 Chrome/Chromium
- **日报为空**：建议先运行 `node fetch.js` 确认抓取正常
- **LLM 调用失败**：自动重试 3 次，全部失败则报告错误详情
